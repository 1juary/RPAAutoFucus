# WaterRPA 快速参考指南 & FAQ

## 🚀 快速参考

### 启动应用
```bash
python -m water_rpa.main
```

### 安装依赖
```bash
pip install -r requirements.txt
```

```json
[
  {
    "type": 1.0,
    "value": "D:\\path\\to\\image.png",
    "retry": 1
  }
]
```

## ⚙️ 重试策略说明

```
retry = 1    ← 尝试一次，失败则放弃
retry = -1   ← 无限重试（直到超时60秒）
retry = 3    ← 重试3次（此时会点击3次）
retry = 5    ← 重试5次
```

**选择建议**:
- 图像识别不稳定 → 使用 `-1` (无限重试，有超时保护)
- 快速单次操作 → 使用 `1`
- 已知需要多次点击 → 使用 `>1` 的数字

## ❌ 常见问题与解决方案

### ❓ Q1: 提示"未找到匹配图片"

**可能原因:**
1. 目标窗口被遮挡或最小化
2. 图像文件不存在或路径错误
3. 目标界面与截图时不同（如分辨率、缩放）
4. 图像识别阈值太高（confidence=0.9）

**解决方案:**
```
步骤1: 确保目标应用窗口在前台
步骤2: 验证图像文件路径是否正确
步骤3: 重新截取一份清晰的目标图像
步骤4: 如仍未解决，尝试修改timeout值（增加等待时间）
```

---

### ❓ Q2: 操作执行超时（60秒）

**可能原因:**
1. 图像识别失败（未找到目标）
2. 系统响应缓慢
3. 网络延迟导致页面加载慢

**解决方案:**
```
步骤1: 在超时前添加等待操作
       例如: 加入 {"type": 5.0, "value": "5.0"}

步骤2: 检查图像识别是否正常
       - 使用清晰的高分辨率截图
       - 确保目标元素在屏幕可见区域

步骤3: 增加重试次数
       - 改用 "retry": -1 (无限重试)

步骤4: 修改源码超时时间 (仅高级用户)
       - 在 mouseClick 函数中修改 timeout 参数
```

---

### ❓ Q3: 文本输入乱码

**可能原因:**
1. 系统输入法问题
2. 目标应用不支持中文输入
3. 编码格式不一致

**解决方案:**
```
步骤1: 确保系统输入法设置正确
       - 切换到中文输入法

步骤2: 若目标应用不支持中文，使用备选方案:
       - 将文本复制到剪贴板
       - 使用 Ctrl+V 粘贴

步骤3: 检查JSON文档编码
       - 确保保存为 UTF-8 格式
```

---

### ❓ Q4: 循环执行无法停止

**可能原因:**
1. 未正确点击[停止]按钮
2. 按钮未响应

**解决方案:**
```
步骤1: 点击[停止]按钮
步骤2: 若无反应，按 Ctrl+C 中断程序
步骤3: 重启应用
步骤4: 仅执行一次，避免循环模式
```

---

### ❓ Q5: 截图保存失败

**可能原因:**
1. 指定目录不存在
2. 目录路径权限不足
3. 磁盘空间不足

**解决方案:**
```
步骤1: 确保目录存在
       - 若不存在，手动创建目录

步骤2: 检查目录权限
       - 右键 → 属性 → 检查读写权限

步骤3: 确保磁盘有足够空间
       - 清理磁盘或指定其他位置

步骤4: 使用完整路径而非相对路径
       - 错误: "Screenshots"
       - 正确: "D:\\Screenshots" 或 "C:\\Users\\Username\\Pictures"
```

---

### ❓ Q6: 最小化窗口后无法识别图像

**原因:**
被最小化的窗口无法进行屏幕截取和图像识别。

**解决方案:**
```
方案1: 不勾选"运行时最小化"选项 (推荐)

方案2: 修改任务队列，在最后恢复窗口
      - 添加任务: {"type": 7.0, "value": "alt+tab"}

方案3: 不使用最小化选项，操作完成后再最小化
```

---

### ❓ Q7: 导入配置文件时提示格式错误

**可能原因:**
1. JSON格式不规范
2. 文件编码不是UTF-8
3. 缺少必含字段

**解决方案:**
```
步骤1: 使用在线JSON验证工具检查格式
       - jsonchecker.com
       - jsonlint.com

步骤2: 确保文件编码为UTF-8
       - 用记事本打开 → 另存为 → 选择UTF-8编码

步骤3: 确保每条记录包含所有字段
       ✓ 正确格式:
       [
         {"type": 1.0, "value": "path", "retry": 1}
       ]
       
       ✗ 错误格式:
       [
         {"type": 1.0}  ← 缺少value和retry
       ]
```

---

### ❓ Q8: 部分快捷键无法执行

**常见快捷键格式:**
```
ctrl+c          ← 复制
ctrl+v          ← 粘贴
ctrl+z          ← 撤销
alt+f4          ← 关闭窗口
alt+tab         ← 切换窗口
shift+tab       ← 反向制表符
Return / Enter  ← 回车
Escape          ← Esc
Delete          ← 删除
Backspace       ← 退格
```

**解决方案:**
```
步骤1: 确保快捷键格式正确（小写）

步骤2: 测试快捷键是否在目标应用有效
       - 手动测试该快捷键

步骤3: 某些快捷键可能与系统冲突
       - 尝试更换快捷键或修改系统设置

步骤4: 若快捷键完全不工作，尝试用鼠标点击替代
```

## 🔍 调试技巧

### 调试技巧1: 逐步执行

```
不要一次导入所有任务，改为:
1. 先执行前2条指令，检查是否正常
2. 再添加第3条，执行检查
3. 依次添加其他指令

这样便于定位出问题的具体步骤
```

### 调试技巧2: 使用截图保存测试

```json
{
  "type": 9.0,
  "value": "D:\\TestScreenshots",
  "retry": 1
}
```

在关键步骤加入截图，查看执行时的实际界面状态。

### 调试技巧3: 充分使用等待操作

```json
{
  "type": 5.0,
  "value": "2.0",
  "retry": 1
}
```

在不稳定的步骤前加入等待，给系统充足时间响应。

### 调试技巧4: 检查日志输出

仔细阅读运行日志，找出是哪一步失败。日志会显示：
- `执行步骤 X: 类型=Y, 内容=Z`
- `单击左键: path`
- `未找到匹配图片, 0.1秒后重试`
- `等待图片 xxx 超时 (60秒)`

## 📊 性能优化建议

### 1. 图像识别优化

```
✓ 推荐做法:
  - 使用清晰的高分辨率截图 (800x600以上)
  - 截取的目标图像应包含足够的特征
  - 避免纯色或重复图案的图像

✗ 避免做法:
  - 模糊或压缩过度的图像
  - 尺寸太小 (<50px) 的图像
  - 部分被遮挡的图像
```

### 2. 重试策略优化

```
✓ 图像识别操作:
  使用 retry = -1 (无限重试，有超时保护)
  
✓ 稳定操作 (如输入文本):
  使用 retry = 1 (一次执行)
  
✓ 需要多次点击:
  使用 retry = 3 或更多
```

### 3. 序列优化

```
不优化:
1. 点击搜索框      (立即执行)
2. 输入文本        (可能失败，焦点未获取)
3. 按回车          (可能不起作用)

优化后:
1. 等待 0.5 秒    (确保焦点获取)
2. 点击搜索框      (重新获取焦点)
3. 等待 0.2 秒    (焦点稳定)
4. 输入文本        (文本输入成功率更高)
5. 等待 0.2 秒    (确保输入完成)
6. 按回车          (确保被捕获)
7. 等待 2.0 秒    (页面加载)
```
