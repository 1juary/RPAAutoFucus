## 📋 项目概述

WaterRPA 是一个基于Python和PySide6开发的**图形化RPA（机器人流程自动化）工具**。它允许用户通过简单的GUI界面配置和执行自动化任务，而无需编写代码。

### 核心特点
- ✅ **无需编程** - 全图形化界面，傻瓜式操作
- ✅ **功能丰富** - 支持9种不同的自动化操作
- ✅ **配置保存** - 支持JSON格式配置文件导入/导出
- ✅ **智能重试** - 内置图像识别重试机制，支持超时控制
- ✅ **循环执行** - 支持单次执行和循环执行模式
- ✅ **实时日志** - 运行过程中实时输出执行日志
- ✅ **任务管理** - 图形编辑任务序列，随意增删和调整

## 🚀 快速开始

### 系统要求
- **Python版本**: 3.10+
- **操作系统**: Windows (支持Win10、Win11)
- **内存**: 最低2GB
- **磁盘空间**: 最低500MB

### 安装步骤

#### 1️⃣ 配置Python环境
```bash
# 创建虚拟环境（推荐）
python -m venv .venv

# 激活虚拟环境
# Windows:
.venv\Scripts\activate

# 或者 PowerShell:
.\.venv\Scripts\Activate.ps1
```

#### 2️⃣ 安装依赖库
```bash
pip install -r requirements.txt
```

> 📌 **首次使用提示**: 首次运行时PyAutoGUI会自动扫描系统，请保持耐心，不要中断安装过程。

#### 3️⃣ 启动应用
```bash
python waterRPA_GUI.py
```

## 💡 功能特性

### 支持的操作类型（9种）

| 操作编码 | 操作名称 | 参数示例 | 应用场景 |
|---------|--------|--------|--------|
| 1.0 | **左键单击** | `button.png` | 点击网页按钮、菜单项等 |
| 2.0 | **左键双击** | `icon.png` | 打开文件、启动应用 |
| 3.0 | **右键单击** | `file.png` | 右键菜单操作 |
| 4.0 | **输入文本** | `Hello World` | 填充表单、搜索框 |
| 5.0 | **等待(秒)** | `2.5` | 等待页面加载、系统响应 |
| 6.0 | **滚轮滑动** | `5` 或 `-5` | 上下滚动网页内容 |
| 7.0 | **系统按键** | `ctrl+c`, `alt+tab` | 复制粘贴、切换窗口 |
| 8.0 | **鼠标悬停** | `target.png` | 鼠标移动到特定位置 |
| 9.0 | **截图保存** | `D:\Screenshots` | 保存屏幕快照、生成报告 |

### 高级特性

#### 🔄 重试机制
- **1** = 尝试一次后放弃
- **-1** = 无限重试（直到找到或超时）
- **n** = 重试n次（n > 1）
- **超时保护** = 默认60秒超时，防止卡死

#### 🔁 执行模式
- **执行一次** = 顺序执行完所有任务后停止
- **循环执行** = 不断重复执行任务序列（按Ctrl+C停止）

#### ⚙️ 运行选项
- **运行时最小化** = 任务运行时自动最小化窗口，完成后自动还原

### 代码模块说明

#### 1. **核心引擎层** (Core Engine)

**`RPAEngine` 类**
- 职责: 任务执行的核心逻辑，状态管理
- 核心方法:
  - `run_tasks()` - 执行任务列表，支持循环和中止
  - `stop()` - 停止任务执行
- 属性:
  - `is_running` - 表示是否正在运行
  - `stop_requested` - 中止请求标志

**`mouseClick()` 函数**
- 职责: 处理鼠标点击操作和图像识别
- 参数说明:
  - `clickTimes` - 点击次数（1单击，2双击）
  - `lOrR` - 按键类型（"left"左键，"right"右键）
  - `img` - 图像路径（使用PyAutoGUI进行识别）
  - `reTry` - 重试策略（1一次，-1无限，>1指定次数）
  - `timeout` - 超时时间（默认60秒）

**`mouseMove()` 函数**
- 职责: 鼠标悬停操作
- 用途: 触发鼠标悬停效果（如菜单展开）

#### 2. **UI层** (User Interface)

**`RPAWindow` 类** (主窗口)
- 布局结构:
  ```
  ┌─ 顶部控制栏 ─────────────────────┐
  │ [+新增] [保存] [导入] 循环□ [开始] [停止]
  ├─────────────────────────────────┤
  │           任务列表区域（滚动）     │
  │  ┌──────────────────────────┐   │
  │  │ TaskRow 1                │   │
  │  ├──────────────────────────┤   │
  │  │ TaskRow 2                │   │
  │  ├──────────────────────────┤   │
  │  │ TaskRow 3                │   │
  │  └──────────────────────────┘   │
  ├─────────────────────────────────┤
  │  运行日志:                       │
  │  [实时日志输出区域]              │
  └─────────────────────────────────┘
  ```

**`TaskRow` 类** (任务行组件)
- 功能:
  - 操作类型选择（下拉菜单）
  - 参数输入（动态显示不同输入框）
  - 文件选择（图像识别和保存路径）
  - 重试次数设置
  - 删除按钮
- UI动态性: 根据选中的操作类型，动态显示/隐藏相关输入框

#### 3. **线程层** (Threading)

**`WorkerThread` 类** (后台工作线程)
- 继承: `QThread`
- 信号:
  - `log_signal` - 发送日志消息
  - `finished_signal` - 任务完成信号
- 作用: 在后台独立线程执行任务，防止UI卡顿

## 📊 运行逻辑流程

### 总体执行流程

```
启动应用
  │
  ▼
用户配置任务序列
  │
  ├─ 添加指令 (TaskRow添加)
  ├─ 设置参数
  ├─ 选择重试策略
  │
  ▼
[保存/导入配置]
  │
  ├─ 保存: 导出为JSON文件
  └─ 导入: 从JSON文件读取配置
  │
  ▼
点击[开始运行]
  │
  ├─ 验证配置合法性
  ├─ 创建后台线程 (WorkerThread)
  ├─ 最小化窗口（如勾选）
  │
  ▼
WorkerThread执行任务
  │
  ├─ 循环执行任务列表
  │
  ├─ 对每个任务:
  │  ├─ 检查中止信号
  │  ├─ 根据操作类型分发
  │  ├─ 执行对应操作
  │  ├─ 发送日志信号
  │  └─ 继续下一个任务
  │
  ├─ 是否循环: 
  │  ├─ 是 → 继续下一轮循环
  │  └─ 否 → 执行完后停止
  │
  ▼
任务完成
  │
  ├─ 发送finished信号
  ├─ 恢复窗口
  └─ 重新启用按钮
```

### 单个任务执行流程

```
获取任务 Task = {type, value, retry}
  │
  ▼
├─ type == 1.0  ──► mouseClick(1, 'left', value, retry)
├─ type == 2.0  ──► mouseClick(2, 'left', value, retry)
├─ type == 3.0  ──► mouseClick(1, 'right', value, retry)
├─ type == 4.0  ──► pyperclip.copy(value) + Ctrl+V
├─ type == 5.0  ──► time.sleep(float(value))
├─ type == 6.0  ──► pyautogui.scroll(int(value))
├─ type == 7.0  ──► pyautogui.hotkey(*keys)
├─ type == 8.0  ──► mouseMove(value, retry)
└─ type == 9.0  ──► pyautogui.screenshot(path)
  │
  ▼
发送日志信号
  │
  ▼
任务完成，继续下一个
```

### mouseClick() 图像识别与重试机制

```
mouseClick(1, 'left', 'button.png', retry=-1, timeout=60)
  │
  ▼
开始时间记录
  │
  ▼
Retry == 1 (一次模式)
  │
  └─► while True:
      ├─ 检查超时 (当前时间 - 开始时间 > timeout)
      │  └─ 超时 → 返回，结束
      │
      ├─ 尝试识别图像
      │  ├─ pyautogui.locateCenterOnScreen('button.png', confidence=0.9)
      │  │
      │  ├─ 找到 (location != None)
      │  │  ├─ 执行点击 pyautogui.click(x, y, clicks=1)
      │  │  └─ break (退出循环)
      │  │
      │  └─ 未找到 (ImageNotFoundException)
      │     └─ 继续重试
      │
      └─ 等待100毫秒后重试
```

## 📖 使用教程

### 基础操作流程

#### 步骤1: 创建任务序列

1. **启动应用** - 双击运行 `waterRPA_GUI.py`
2. **添加指令** - 点击 `[+ 新增指令]` 按钮
3. **配置参数** - 对每一行进行配置：
   - 从下拉菜单选择操作类型
   - 输入对应参数
   - 如果涉及图像，点击 `[选择图片]` 浏览文件
   - 设置重试次数（如需要）

#### 步骤2: 验证并保存配置

1. **检查配置** - 确保所有参数都已正确填写
2. **保存配置** - 点击 `[保存配置]` 按钮
3. **选择位置** - 选择保存位置，设置文件名（如 `my_task.json`）

#### 步骤3: 运行任务

1. **选择执行模式** - 从下拉菜单选择"执行一次"或"循环执行"
2. **勾选选项** - 根据需要勾选"运行时最小化"
3. **点击开始** - 点击 `[开始运行]` 按钮
4. **观察日志** - 在日志区域实时查看执行情况

#### 步骤4: 中止任务

- 点击 `[停止]` 按钮立即中止任务执行
- 或等待任务自然完成

## ⚠️ 注意事项

### 🔒 安全提示

1. **图像识别精度** - 确保待识别的图像清晰且分辨率不失真
2. **超时设置** - 根据系统性能调整超时时间，避免卡死
3. **路径处理** - 使用绝对路径指定图像和保存位置，避免路径错误
4. **备份配置** - 重要的任务配置应定期备份

### 🚨 常见问题排查

#### 问题1: "未找到匹配图片"

**原因分析:**
- 目标窗口被遮挡或最小化
- 图像识别失败（图像清晰度不足，confidence阈值过高）
- 路径不正确

**解决方案:**
- 确保目标应用在前台运行
- 用截图软件重新截取清晰的目标图像
- 检查图像文件路径是否正确

#### 问题2: "操作超时"

**原因分析:**
- 系统响应缓慢
- 网络延迟（网页加载慢）
- 超时时间设置过短

**解决方案:**
- 增加等待时间（使用`5.0`类型操作）
- 优化系统性能或网络环境
- 增加默认超时（修改源码timeout参数）

#### 问题3: 窗口最小化后无法找到图像

**原因:**
- 最小化的应用无法执行图像识别

**解决方案:**
- 不勾选"运行时最小化"选项
- 在任务最后添加启用最小化逻辑

#### 问题4: 某些快捷键无法正确执行

**原因:**
- 快捷键格式错误
- 快捷键与系统冲突

**解决方案:**
- 使用正确的快捷键语法（如`ctrl+c`、`alt+f4`）
- 测试快捷键是否在目标应用中有效
